# Rúbrica de Simulacros

## Uso

Copiar esta plantilla después de cada simulacro y completar la evaluación en `notes/review-queue.md`.

## Plantilla

```markdown
## Simulacro YYYY-MM-DD

- Problema:
- Patrón esperado:
- Tiempo total:
- Resultado:
- Tests escritos:
- Complejidad explicada:

### Puntaje

| Dimensión | Puntaje 1-5 | Evidencia |
| --- | --- | --- |
| Clarificación |  |  |
| Enfoque |  |  |
| Implementación |  |  |
| Pruebas |  |  |
| Comunicación |  |  |

### Retro

- Qué salió bien:
- Qué falló:
- Error que se repite:
- Acción de repaso:
- Fecha para repetir:
```

## Criterios

Clarificación:

- 1: codifica sin confirmar restricciones.
- 3: confirma ejemplos y tipo de entrada.
- 5: detecta casos borde antes de diseñar.

Enfoque:

- 1: elige patrón por intuición sin justificar.
- 3: explica patrón y complejidad.
- 5: compara brute force, solución óptima y trade-offs.

Implementación:

- 1: solución incompleta o con errores estructurales.
- 3: solución correcta con correcciones durante la marcha.
- 5: solución clara, idiomática y estable.

Pruebas:

- 1: solo prueba el caso feliz.
- 3: cubre caso normal y borde.
- 5: cubre borde, negativo y regresión del error cometido.

Comunicación:

- 1: silencios largos o saltos sin explicar.
- 3: explica pasos principales.
- 5: narra decisiones, invariantes y riesgos con orden.
