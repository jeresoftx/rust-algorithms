# Semanas 02 y 03: Arrays, Strings, Hashing y Sumas de Prefijos

## Patrones Trabajados

### HashMap para Complementos

El mapa guarda información de valores anteriores. En `two_sum`, si `target - value` ya existe, encontramos una pareja válida.

Invariante:

- Antes de procesar el índice actual, el mapa solo contiene índices anteriores.
- Esto evita reutilizar el mismo elemento dos veces.

### Conteo de Frecuencias

Se usa cuando el problema depende de cuántas veces aparece un valor o carácter.

Ejemplos:

- `valid_anagram`
- `top_k_frequent`
- `group_anagrams`
- `first_unique_char`

Pregunta guía:

- ¿Necesito conteos exactos, solo presencia o agrupación?

### Mapeo Bidireccional

Se usa cuando dos dominios deben mantener una relación uno a uno.

Ejemplos:

- `is_isomorphic`
- `word_pattern`

Invariante:

- Si un carácter ya tiene destino, debe volver a ver el mismo destino.
- Si un destino ya fue usado por otro origen, la relación deja de ser biyectiva.

### Llave Canónica

Se usa cuando varias entradas distintas deben caer en el mismo grupo.

Ejemplo:

- `"eat"`, `"tea"` y `"ate"` se normalizan como `"aet"`.

Compensación:

- Ordenar cada palabra es simple y claro.
- Para alfabetos limitados, una llave de conteos puede mejorar la complejidad.

### Prefijo/Sufijo

`product_except_self` usa dos pasadas:

- Una para guardar el producto acumulado antes de cada índice.
- Otra para multiplicar por el producto acumulado después de cada índice.

Este patrón evita división y maneja ceros correctamente.

### Suma de Prefijos con HashMap

`subarray_sum_equals_k` cuenta subarreglos contiguos cuya suma es `k`.

Invariante:

- `prefix_sum` representa la suma acumulada hasta el índice actual.
- Si `prefix_sum - k` apareció antes, existe un subarreglo que termina en el índice actual y suma `k`.
- El mapa guarda cuántas veces apareció cada suma de prefijos.

## Errores Comunes

- Olvidar inicializar el mapa de sumas de prefijos con `{0: 1}`.
- Confundir subarreglo contiguo con subsecuencia.
- Ordenar resultados de un `HashMap` sin recordar que el orden no es estable.
- Usar `HashMap` cuando un `HashSet` basta.
- No probar números negativos en problemas de sumas acumuladas.
- Validar solo un lado del mapeo y permitir que dos claves apunten al mismo valor.
